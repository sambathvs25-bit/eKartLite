• Run & test checklist:

  1. Start infra2:

  docker compose up -d mysql redis

  2. Build once:

  mvn -s mvn-settings.xml clean compile -DskipTests

  3. Start all services (new terminal):

  powershell -ExecutionPolicy Bypass -File .\scripts\start-all.ps1 -SkipBuild

  4. Check health (new terminal):

  curl http://localhost:8080/actuator/health
  curl http://localhost:8081/actuator/health
  curl http://localhost:8082/actuator/health
  curl http://localhost:8083/actuator/health
  curl http://localhost:8084/actuator/health
  curl http://localhost:8085/actuator/health
  curl http://localhost:8086/actuator/health
  curl http://localhost:8087/actuator/health

  5. Quick API test:

  - Register customer: POST /auth/register
  - Register agent: POST /auth/register-agent
  - Login both: POST /auth/login
  - Add product (agent token): POST /agent/products
  - Search (customer token): GET /customer/products/search
  - Add cart item: POST /customer/cart/items
  - Create order: POST /customer/orders
  - Pay order: POST /customer/orders/{id}/pay
  - Check shipping orders (agent): GET /agent/orders/ready-for-shipping

  6. Stop all:

  powershell -ExecutionPolicy Bypass -File .\scripts\stop-all.ps1
  docker compose down

  If any step fails, share that command + error and I’ll fix immediately.

  • 1. Start infrastructure first

  - Open terminal in project root D:\Ecommerce_project
  - Run:

  docker compose up -d mysql redis

  2. Import project in Eclipse/STS

  - File -> Import -> Maven -> Existing Maven Projects
  - Select D:\Ecommerce_project
  - Ensure all modules are imported (api-gateway, auth-service, customer-service, product-service, cart-service, order-service,
    payment-service, batch-service)

  3. Enable annotation processing (important)

  - Project -> Properties -> Java Compiler -> Annotation Processing
  - Check Enable project specific settings and Enable annotation processing
  - Apply for each module (or all projects)

  4. Update Maven dependencies

  - Right click parent project -> Maven -> Update Project (Alt+F5)
  - Select all modules and force update

  5. Build once

  - Project -> Clean... then build all
  - Wait for no compile errors

  6. Run services in this order (Run As -> Spring Boot App)
  7. auth-service (AuthServiceApplication)
  8. payment-service (PaymentServiceApplication)
  9. order-service (OrderServiceApplication)
  10. product-service (ProductServiceApplication)
  11. cart-service (CartServiceApplication)
  12. customer-service (CustomerServiceApplication)
  13. batch-service (BatchServiceApplication)
  14. api-gateway (ApiGatewayApplication)
  15. Verify ports

  - 8081, 8086, 8085, 8083, 8084, 8082, 8087, 8080 should be up

  8. Smoke test

  - Open browser/Postman:
      - http://localhost:8080/actuator/health
      - http://localhost:8081/actuator/health

  If any module fails in Eclipse, send the first Caused by: block from Console and I’ll give exact fix.
